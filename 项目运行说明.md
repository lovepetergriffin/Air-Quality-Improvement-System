# NEP环保公众监督系统运行说明

## 项目结构

```
项目源码/
├── 01.前端/
│   ├── neps   公众监督员移动端，端口 8081
│   ├── nepg   网格员移动端，端口 8083
│   └── nepm   管理员后台端，端口 8082
├── 02.后端/
│   └── nep-backend  Spring Boot 后端，端口 8080
└── 03.数据库/
    ├── nep.sql
    └── nep_fixed.sql
```

## 数据库

1. 创建 MySQL 数据库 `nep`。
2. 导入 `项目源码/03.数据库/nep.sql`，如遇到兼容问题再导入 `nep_fixed.sql`。
3. 后端数据库配置在：

```
项目源码/02.后端/nep-backend/src/main/resources/application.yml
```

默认用户名为 `root`，数据库密码请通过环境变量 `NEP_DB_PASSWORD` 配置，或在本地运行时按自己的 MySQL 密码修改。

## 启动后端

```bash
cd 项目源码/02.后端/nep-backend
mvn spring-boot:run
```

后端地址：

```
http://localhost:8080
```

## 启动前端

公众监督员端：

```bash
cd 项目源码/01.前端/neps
```

## 业务流程

1. 公众监督员在 `neps` 注册或登录，选择省市和详细地址，选择预估 AQI，提交污染反馈。
2. 管理员在 `nepm` 查看公众反馈数据，查看详情后指派本地或异地网格员。
3. 网格员在 `nepg` 登录后查看待处理任务，录入 SO2、CO、PM2.5 实测数据，系统计算综合 AQI 并提交。
4. 管理员在 `nepm` 查看确认 AQI 数据和统计分析结果。
