# NEP 环保公众监督系统使用说明书

## 1. 文档说明

本文档用于说明 NEP 环保公众监督系统的运行方式、访问方式、账号说明、业务流程和常见使用场景。系统包含公众监督员移动端、网格员移动端、管理员后台端和统一后端服务。

本说明书适用于以下场景：

- 本机启动项目并演示完整业务流程。
- 局域网内让其他同学通过手机或电脑访问系统。
- 公众监督员提交环境污染反馈。
- 管理员查看反馈、指派网格员、查看 AQI 确认数据和统计结果。
- 网格员查看待处理任务并录入实测污染物数据。
- 排查端口占用、后端连接失败、任务列表无数据等问题。

## 5. 局域网访问说明

### 5.1 适用场景

如果老师或同学想用自己的手机访问你的项目，需要满足以下条件：

- 你的电脑和对方手机在同一个局域网或校园网内。
- 你的后端和对应前端已经启动。
- Windows 防火墙没有拦截 `8080`、`8081`、`8082`、`8083` 端口。
- 前端服务是通过 `npm run dev` 启动的，因为当前脚本已经配置 `--host 0.0.0.0`。

### 5.2 查看本机局域网 IP

在终端执行：

```bash
ipconfig
```

找到无线网卡或有线网卡中的 IPv4 地址，例如：

```text
10.17.112.79
```

### 5.3 局域网访问地址

假设你的电脑 IP 是 `10.17.112.79`，则局域网内其他设备访问：

```text
公众监督员端：http://10.17.112.79:8081
管理员后台端：http://10.17.112.79:8082
网格员端：http://10.17.112.79:8083
后端服务：http://10.17.112.79:8080
```

当前三个前端已经配置为自动使用访问页面时的主机 IP 请求后端。例如手机访问：

```text
http://10.17.112.79:8081
```

前端会自动请求：

```text
http://10.17.112.79:8080
```

所以不需要再把前端代码里的接口地址手动改成固定 IP。

### 5.4 手机访问失败时检查

如果手机打不开页面，按下面顺序检查：

1. 确认电脑和手机连的是同一个网络。
2. 确认前端服务终端没有关闭。
3. 确认访问的是电脑的 IPv4 地址，不是 `localhost`。
4. 确认 Windows 防火墙没有拦截 Node.js 或 Java。
5. 在电脑浏览器中先访问 `http://10.17.112.79:8081`，确认电脑自己能通过局域网 IP 打开。
6. 如果微信或 QQ 内置浏览器缓存旧页面，在地址后面加参数刷新，例如：

```text
http://10.17.112.79:8081/?v=1
```

## 7. 业务状态说明

公众反馈数据大致有三种状态：

| 状态值 | 含义 | 出现场景 |
| --- | --- | --- |
| `0` | 未指派 | 公众刚提交反馈，管理员还没有安排网格员 |
| `1` | 已指派 / 待检测 | 管理员已经将反馈指派给某个网格员 |
| `2` | 已处理 / 已确认 | 网格员已经录入实测数据并提交确认 |

网格员端只显示状态为 `1` 且属于当前登录网格员的任务。任务完成后状态变为 `2`，因此不再出现在待处理任务列表中。

## 8. 完整业务流程

系统完整业务闭环如下：

```text
公众监督员注册或登录
-> 选择省份、城市和详细地址
-> 选择预估 AQI 等级并填写污染描述
-> 提交污染反馈
-> 管理员登录后台
-> 管理员查看反馈详情
-> 管理员将反馈指派给本地或异地网格员
-> 网格员登录移动端
-> 网格员查看待处理任务
-> 网格员查看反馈详情并录入 SO2、CO、PM2.5
-> 系统计算或匹配确认 AQI
-> 网格员提交确认数据
-> 反馈状态变为已处理
-> 管理员查看确认 AQI 数据和统计分析
```

## 9. 公众监督员端使用说明

公众监督员端访问地址：

```text
http://localhost:8081
```

局域网访问示例：

```text
http://10.17.112.79:8081
```

### 9.1 登录

适用场景：已有公众监督员账号，需要提交反馈或查看历史反馈。

操作步骤：

1. 打开公众监督员端。
2. 在登录页输入手机号。
3. 输入密码。
4. 点击登录按钮。
5. 登录成功后进入公众监督员首页或上报页面。

示例账号：

```text
手机号：13147859658
密码：123
```

如果提示无法连接后端服务：

- 确认后端 `8080` 已启动。
- 确认手机访问时使用的是电脑 IP。
- 确认防火墙没有拦截 `8080`。

### 9.2 注册

适用场景：没有公众监督员账号，需要创建新账号。

操作步骤：

1. 在登录页点击注册入口。
2. 输入手机号。
3. 输入姓名。
4. 选择性别或填写基本信息。
5. 输入密码。
6. 确认注册。
7. 注册成功后返回登录页。
8. 使用刚注册的手机号和密码登录。

注意事项：

- 手机号不能和已有账号重复。
- 密码不要留空。
- 如果提示手机号已存在，说明该手机号已经注册，可以直接登录。

### 9.3 提交污染反馈

适用场景：公众发现空气污染情况，需要向系统提交反馈。

操作步骤：

1. 登录公众监督员端。
2. 进入上报页面。
3. 选择省份。
4. 根据省份选择城市。
5. 填写详细地址，例如街道、小区、路段或厂区附近。
6. 点击下一步，进入 AQI 选择页面。
7. 选择自己预估的 AQI 等级。
8. 填写污染描述，例如异味、烟尘、雾霾、扬尘等情况。
9. 点击提交。
10. 系统提示提交成功后，反馈进入管理员待处理列表。

填写建议：

- 地址尽量具体，方便管理员判断是否本地指派。
- 污染描述尽量说明污染现象、时间和影响。
- AQI 等级是公众预估值，后续会由网格员实测确认。

### 9.4 查看我的反馈

适用场景：公众监督员想查看自己提交过哪些反馈，以及处理进度。

操作步骤：

1. 登录公众监督员端。
2. 点击底部导航中的“我的反馈”。
3. 查看反馈列表。
4. 根据状态判断处理进度。

常见状态含义：

- 未指派：管理员还未安排网格员。
- 已指派：管理员已经安排网格员处理。
- 已处理：网格员已经完成检测并提交确认数据。

### 9.5 查看个人信息和退出登录

适用场景：确认当前登录账号或切换账号。

操作步骤：

1. 登录公众监督员端。
2. 点击底部导航中的“个人”。
3. 查看手机号、姓名等信息。
4. 如果需要切换账号，点击退出登录。
5. 返回登录页后输入其他账号登录。

适用场景：管理员需要查看公众提交的所有污染反馈。

1. 进入公众监督数据列表。
2. 找到目标反馈。

4. 查看反馈地址、污染描述、预估 AQI、提交人和提交时间。
5. 根据详情决定是否指派网格员处理。

### 10.4 指派本地网格员

适用场景：反馈所属省市有可用网格员，管理员将任务交给当地网格员处理。

操作步骤：

1. 进入公众监督数据列表。
2. 找到未指派的反馈。
3. 点击指派按钮。
4. 进入指派页面。
5. 选择本地指派。
6. 在网格员列表中选择一个可用网格员。
7. 点击确认指派。
8. 系统保存指派信息。
9. 该反馈状态变为已指派。
10. 对应网格员登录网格员端后，可以在任务列表中看到该任务。

注意事项：

- 如果目标省市没有可用网格员，可以使用异地指派。
- 已处理的反馈通常不需要再次指派。

### 10.5 指派异地网格员

适用场景：本地没有合适网格员，或需要临时安排其他地区网格员处理。

操作步骤：

1. 进入公众监督数据列表。
2. 找到需要处理的反馈。
3. 点击指派。
4. 选择异地指派。
5. 选择网格员所在省份。
6. 选择网格员所在城市。
7. 选择具体网格员。
8. 点击确认指派。
9. 系统保存指派信息。
10. 被指派的网格员登录后即可看到任务。

### 10.6 查看确认 AQI 数据列表

适用场景：管理员需要查看网格员提交的实测污染物数据和确认结果。

操作步骤：

1. 登录管理员后台。
2. 在左侧菜单进入确认 AQI 数据列表。
3. 查看已确认数据表格。
4. 可以按分页查看更多数据。
5. 如果页面提供筛选条件，可以按省份、城市或日期查询。

确认数据通常包括：

- 原始反馈编号。
- 网格员信息。
- 省份、城市和地址。
- SO2 实测值。
- CO 实测值。
- PM2.5 实测值。
- 确认 AQI 等级。
- 确认日期和时间。

### 10.7 查看确认 AQI 详情

适用场景：管理员需要查看某条确认数据的完整内容。

操作步骤：

2. 找到目标记录。
3. 点击详情。
4. 查看反馈信息、实测数据、AQI 等级和网格员信息。

### 10.8 查看省份分组统计

适用场景：管理员需要了解各省份反馈或污染确认数据的分布情况。

操作步骤：

1. 登录管理员后台。
2. 在左侧菜单进入统计数据管理。
3. 点击省份分组检查统计。
4. 查看各省份统计表格。
5. 根据数量判断重点关注区域。

### 10.9 查看 AQI 分布统计

适用场景：管理员需要了解不同 AQI 等级出现的数量。

操作步骤：

1. 登录管理员后台。
2. 进入 AQI 指数分布统计页面。
3. 查看图表或表格。
4. 对比优、良、轻度污染、中度污染、重度污染、严重污染等等级数量。

用途：

- 判断整体空气质量情况。
- 发现高污染等级占比是否偏高。
- 为后续管理和分析提供依据。

### 10.10 查看 AQI 趋势统计

适用场景：管理员需要观察 AQI 数据随时间变化趋势。

操作步骤：

1. 登录管理员后台。
2. 进入 AQI 指数趋势统计页面。
3. 选择或查看默认统计维度。
4. 查看折线图或趋势表格。
5. 判断污染情况是否上升、下降或保持平稳。

### 10.11 查看其他统计指标

适用场景：管理员需要查看系统整体指标。

操作步骤：

1. 登录管理员后台。
2. 进入其他数据统计页面。
3. 查看系统提供的统计卡片或指标。

### 11.5 任务完成后列表显示 No Data

- 已处理任务不再属于待处理列表。
- 如果该网格员没有其他待处理任务，就会显示 `No Data`。

5. 当前网格员重新登录或刷新任务列表。
6. 新任务会显示出来。

## 13. 常见问题处理

### 13.1 后端启动提示 8080 被占用

原因：已经有程序占用了 `8080`，可能是之前启动的后端还在运行。

检查命令：

```bash
netstat -ano | findstr :8080
```

如果看到类似：

```text
TCP 0.0.0.0:8080 LISTENING 33232
```

说明 `33232` 是占用端口的进程 PID。

结束进程：

```bash
taskkill /PID 33232 /F
```

然后重新启动后端：

```bash
cd D:\大三下Java实训大项目\NEP环保公众监督系统_完整项目代码_20260616\项目源码\02.后端\nep-backend
mvn spring-boot:run
```

### 13.2 前端提示无法连接后端服务

可能原因：

- 后端 `8080` 没有启动。
- 手机访问时没有使用电脑 IP。
- 防火墙拦截了 `8080`。
- 后端启动失败。
- 数据库连接失败导致后端接口异常。

处理步骤：

1. 先在电脑浏览器访问：

```text
http://localhost:8080/aqi/listAqiAll
```

2. 如果本机能访问，再用局域网 IP 访问：

```text
http://10.17.112.79:8080/aqi/listAqiAll
```

3. 如果本机能访问但手机不能访问，重点检查防火墙和网络。
4. 如果本机也不能访问，查看后端终端报错。

### 13.3 管理端或接口出现 HTTP 500

HTTP 500 表示后端运行时出错。

常见原因：

- 数据库连接失败。
- 数据库表缺失。
- 请求参数不完整。
- 当前运行的后端不是最新代码。
- 后端启动后没有重启，旧代码仍在运行。

处理步骤：

1. 查看后端终端完整报错。
2. 确认 MySQL 正常启动。
3. 确认 `application.yml` 中数据库账号密码正确。
4. 确认数据库 `nep` 存在。
5. 停止旧后端进程后重新启动。

### 13.4 网格员端登录后显示 No Data

这通常不是错误。

含义：当前网格员没有待处理任务。

出现原因：

- 管理员还没有给该网格员指派任务。
- 该网格员的任务已经完成。
- 当前网格员账号不是被指派的账号。

解决方式：

1. 用管理员账号登录后台。
2. 找到一条未指派反馈。
3. 指派给当前网格员。
4. 网格员端刷新页面或重新登录。

也可以换用初始数据中有待处理任务的账号，例如：

```text
guanyu / 123
machao / 123
```

### 13.5 完成网格员任务后列表为空

这是正常业务逻辑。

任务提交确认后，反馈状态从已指派变为已处理。网格员端只显示待处理任务，所以已处理任务不会继续显示。

### 13.6 公众端提交后管理员看不到新反馈

处理步骤：

1. 确认公众端提交时提示成功。
2. 管理员反馈列表刷新页面。
3. 检查分页，最新数据通常在前面。
4. 如果使用了筛选条件，先清空筛选。
5. 确认公众端和管理员端连接的是同一个后端。

### 13.7 管理员指派后网格员看不到任务

处理步骤：

1. 确认管理员指派成功。
2. 确认网格员登录的是被指派的账号。
3. 确认反馈状态已经变为已指派。
4. 网格员端刷新页面或重新登录。
5. 如果任务已经被提交处理，则不会出现在待处理列表。

### 13.8 手机访问页面能打开，但登录时报错

这种情况通常说明前端能访问，但后端访问失败。

处理步骤：

1. 手机打开：

```text
http://10.17.112.79:8080/aqi/listAqiAll
```

2. 如果打不开，说明手机访问不到后端。
3. 检查防火墙。
4. 确认后端监听的是 `8080`。
5. 确认电脑和手机在同一网络。

### 13.9 npm install 或 npm run dev 失败

处理步骤：

1. 确认已经进入正确前端目录。
2. 确认 Node.js 和 npm 已安装。
3. 删除 `node_modules` 后重新执行 `npm install`。
4. 如果网络下载依赖失败，切换网络或配置 npm 镜像源。

正确目录示例：

```text
D:\大三下Java实训大项目\NEP环保公众监督系统_完整项目代码_20260616\项目源码\01.前端\neps
D:\大三下Java实训大项目\NEP环保公众监督系统_完整项目代码_20260616\项目源码\01.前端\nepg
D:\大三下Java实训大项目\NEP环保公众监督系统_完整项目代码_20260616\项目源码\01.前端\nepm
```

### 13.10 页面显示旧内容

可能原因：

- 浏览器缓存。
- 前端服务没有重启。
- 手机内置浏览器缓存旧资源。

处理方式：

1. 刷新页面。
2. 地址后加参数：

```text
?v=1
```

3. 关闭浏览器页面重新打开。
4. 重启对应前端服务。
